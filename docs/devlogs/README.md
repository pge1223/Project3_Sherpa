# 개인 Devlog 규칙

담당자별로 각자 작업 로그를 남기는 폴더다. 회의록이나 PR 설명과 달리,
그날그날 무엇을 시도했고 왜 그렇게 결정했는지를 편하게 남기는 곳이다.

## 파일명

```text
docs/devlogs/{이니셜}-devlog.md
```

이니셜은 각자 쓰는 feature 브랜치 이니셜과 동일하게 맞춘다.

| 담당자 | 파일명 |
|---|---|
| 가은 | `pge-devlog.md` |
| 윤한 | `kyh-devlog.md` |
| 경이 | `mky-devlog.md` |
| 용준 | `kyj-devlog.md` |
| 재인 | `lji-devlog.md` |

## 작성 규칙

- **새 항목은 파일 맨 위에 추가한다.** 아래로 갈수록 과거 기록이라, 항상
  최신 로그가 파일을 열자마자 보인다.
- 항목 구분은 날짜 헤더(`## YYYY-MM-DD`)로 한다.
- 항목 템플릿:

  ```markdown
  ## YYYY-MM-DD

  - 한 일:
  - 결정/이유:
  - 막힌 점:
  - 다음 할 일:
  ```

- 하루에 여러 번 커밋해도 로그는 날짜당 하나로 합쳐서 이어 쓴다 (같은
  `## YYYY-MM-DD` 아래에 추가).
- 다른 담당자의 devlog 파일은 읽는 건 자유지만 직접 수정하지 않는다.

## 예시 (날짜가 쌓이는 모습)

새 날짜는 위에 추가하고, 같은 날 안에서는 기존 헤더 아래에 이어 쓴다.

```markdown
## 2026-07-14

- 한 일: ...

## 2026-07-13

- 한 일: ...
```

7월 15일에 새로 쓰면 `## 2026-07-15`가 맨 위, 7월 14일이 그다음, 7월 13일이 그 아래로 내려간다.
