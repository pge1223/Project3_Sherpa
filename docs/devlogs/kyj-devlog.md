# kyj Devlog

## 2026-07-14

- 한 일
  - RAG 문서 파서 구현

    - 파일 형식별 파서와 `UnifiedDocumentParser` 추가
    - 손상 파일, 빈 파일, 미지원 형식 예외 처리 및 테스트 fixture 구성
  - 이미지 OCR 기능 추가
    - OCR 추상 인터페이스와 EasyOCR 엔진 구현
    - PDF 및 PPTX 내부 이미지에서 텍스트를 추출하도록 기존 파서 확장
    - 일반 텍스트 문서와 이미지 기반 문서를 함께 검증하는 테스트 추가
  - URL 기반 공고문·첨부파일 수집 기능 구현
    - 공고 페이지 HTML 파싱 및 PDF/DOCX/PPTX 첨부파일 링크 탐색
    - 첨부파일 다운로드, 크기·형식 검증, URL 로딩 결과 스키마와 예외 정의
    - URL 로더 자동 테스트와 OCR 수동 확인 스크립트 추가
  - 공고문 HTML 정제 기능 개선
    - 내비게이션, 광고, 푸터 등 RAG에 불필요한 영역을 제거하고 제목·본문·목록·표 구조 보존
    - 공고 관련 키워드를 이용해 본문 후보를 판별하고 첨부파일 탐색 정확도 개선
    - URL 조회, 본문 정제, 첨부파일 탐색을 각각 확인하는 수동 검증 스크립트 추가
  - 원문 위치를 보존하는 RAG 문서 청킹 기능 구현
    - PDF 페이지, PPTX 슬라이드, DOCX 문서, 웹 섹션 위치 정보를 청크 메타데이터에 유지
    - 제목·본문·목록·표·목차를 구분하고 표와 본문을 별도 규칙으로 청킹
    - 원본 블록 ID·순서 역매핑, 섹션 제목 상속, 의사 제목 인식, 작은 꼬리 청크 병합 처리
    - 동일 입력과 설정에서 같은 ID가 생성되는 결정적 `chunk_id` 적용
    - 파일 파싱 결과와 정제된 웹 콘텐츠를 공통 청킹 입력으로 변환하는 어댑터 구현
    - 청크 크기, overlap, 표 분할, 목차 제외, 원본 불변성, 위치 보존 등에 대한 테스트 추가

- 결정/이유
  - 파싱·로딩·전처리·청킹 단계를 독립 모듈로 분리해 각 단계를 단독 실행하고 테스트할 수 있도록 구성
  - 청킹 과정에서 입력 객체를 변경하지 않고 어댑터에서 필요한 값만 복사해 기존 모듈과의 결합도를 낮춤
  - 페이지·슬라이드·원본 블록 정보를 청크에 남겨 검색 결과가 실제 문서 위치를 추적할 수 있도록 설계
  - 목차는 검색 대상에서 제외하고 표는 본문과 섞지 않아 검색 품질 저하를 방지
  - 청크 ID에 문서, 위치, 설정, 내용을 반영해 재실행 시 안정성과 변경 감지를 확보

- 막힘 해결
  - OCR이 필요한 이미지형 PDF/PPTX를 별도 fixture와 수동 확인 스크립트로 검증
  - 실제 공고 페이지에서 표의 섹션 제목이 유실되거나 목록이 중간에서 잘리는 문제를 회귀 테스트로 재현해 보완
  - 목차와 유사한 본문을 무조건 제외하지 않고 명확한 목차 제목이 있을 때만 `TOC`로 분류하도록 처리
  - `dev` 브랜치의 변경을 작업 브랜치에 병합하며 충돌을 정리하고 PR 단위로 기능 반영

  - 실제 공고문 표본을 늘려 파서, OCR, HTML 정제, 청킹 회귀 테스트 보강
