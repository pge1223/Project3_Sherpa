AI Review Board · User Journey

공모전 준비, 아이디어부터 완성본까지
한 서비스 안에서

아이디어 하나 없이 막막했던 김나영이, AI Review Board를 거치며 겪는 여정을 그대로 따라가 봅니다.

나
김나영
서울시 공모전에 나가고 싶지만, 아직 좋은 아이디어가 떠오르지 않아 AI Review Board를 찾았다.
ACT I
시작 — 아직 아이디어가 없다
공모전은 정했는데, 무엇을 만들지가 없다
01
공모전 정보 입력

김나영은 나가고 싶은 공모전의 URL을 입력하고, 사이트에 있던 평가기준 문서를 함께 올렸다. 잠시 후 화면에 안내가 떴다 — 공고문을 다 읽었고, 핵심 주제와 평가 기준, 요구 분량까지 파악했다는 내용이었다.

핵심 동작
  • 나가려는 공모전 URL 입력
  • 공모전 사이트의 평가기준 문서나 신청서 양식 업로드
  • 시스템이 공고문을 읽고 핵심 주제 · 공모 분야 · 평가 기준 · 요구 분량을 파악
02
작성 전 모드
아이디어 발굴 회의

"그런데… 무슨 아이디어로 나가지?" 정작 막막한 건 이제부터였다. 진행자가 이끄는 회의실에 들어가자, 기획 전문가와 개발 전문가가 방금 읽은 공고문을 근거로 아이디어를 하나씩 제안했다. 몇 번의 대화 끝에 김나영은 마음에 드는 아이디어를 하나 골랐다.

핵심 동작
  • 아직 기획서를 쓰기 전, 주제 자체가 막막한 단계
  • 진행자가 회의를 이끌고, 기획 · 개발 전문가 2명이 공고문 내용을 근거로 아이디어를 제안
  • 김나영은 여러 아이디어 중 마음에 드는 하나를 선택
03
선택한 아이디어 심화 회의

아이디어를 고르고 나니 이번엔 그걸 구체화할 차례였다. 같은 전문가들과 다시 마주 앉아, 어떤 데이터를 쓸지 · 어떻게 구현할지 · 다른 아이디어와 뭐가 다른지를 하나씩 채워나갔다.

핵심 동작
  • 고른 아이디어를 놓고 전문가들과 추가 회의
  • 데이터 활용 방안 · 기술 구현 방향 · 차별점 등 구체화
ACT II
쓰다 — 회의를 문서로 옮기다
방향은 이미 정해졌다, 이제 정리할 차례
04
기획서 작성

며칠 뒤, 김나영은 회의에서 정리된 아이디어와 근거를 그대로 살려 기획서를 써내려갔다. 이미 방향과 데이터, 차별점까지 다 나온 상태라 백지에서 시작하는 것과는 완전히 달랐다 — 회의 내용을 공모전 양식에 옮겨 담는 것에 가까웠다.

핵심 동작
  • 회의에서 구체화된 내용을 바탕으로, 공모전 양식에 맞춰 기획서 작성
ACT III
성장하다 — 점검하고 심사받으며 조금씩 나아지다
완성이 아니라 계속 좋아지는 과정
05
작성 후 모드
문서 피드백 — AI 피드백 워크벤치

기획서를 다 쓰고 "문서 피드백" 화면에 파일을 올렸다. 몇 초 만에 결과가 떴다 — 오탈자, 흐름이 끊기는 문단, 공고문 기준보다 살짝 모자란 분량, 군데군데 휑한 페이지. 원문 위에 문제 되는 부분이 그대로 표시돼 있어 뭘 고쳐야 할지 한눈에 들어왔다.

핵심 동작
  • 작성한 기획서를 업로드하면 4가지를 자동 점검
  • 오탈자 — 잘못된 표현 탐지
  • 맥락 이상 — 문서 흐름과 안 맞는 문단 탐지
  • 분량 — 공고문 기준 대비 페이지 수 충족 여부
  • 밀도 — 페이지가 실제로 얼마나 채워져 있는지
  • 원문을 페이지 그대로 보여주며, 문제 되는 부분을 하이라이트로 표시
06
개인 맞춤형 피드백 루프
완성 리포트 — 문서 버전 추적형

고친 곳을 마저 손보는 사이, 위원회 심사 결과도 리포트 화면에 함께 올라왔다. 이 결과는 v1.0으로 기록됐고, 기획 위원 · 개발 위원의 채점 옆에는 방금 워크벤치에서 봤던 오탈자 지적이 "AI 피드백" 탭으로 그대로 이어져 있었다.

기획 위원 · 개발 위원
공고문에서 자동 추출한 평가기준 · 배점 그대로 각 항목을 점수 · 판정으로 채점하고, 근거가 된 문서 문장을 인용해 "왜 이 점수인지"까지 보여줌
AI 피드백 탭
STEP 05에서 나온 오탈자 · 맥락 이상 결과가 그대로 들어와, 위원 채점과는 별개로 "수정 필요 / 해결" 상태로만 추적됨 (점수에는 반영 안 됨)

개발 위원이 지적한 부분을 보니, 옆에 구현 난이도 안내가 붙어 있었다. 김나영은 비전공자라 마이페이지에 등록해둔 이력 그대로 "어려울 수 있음 — 단계별로 이렇게 해보세요"라는 안내를 받았다. 회의록과 이 안내를 참고해 기획서를 다시 고치고 재분석을 요청하자, v1.1이 새로 쌓였다. 리포트 화면에는 v1.0과 나란히 점수 추이와 항목별 막대가 떴고, 뭐가 해결됐고 뭐가 아직 남았는지 한눈에 보였다.

버전 비교
v1.0 → 수정 · 재분석 → v1.1 → ...
해결된 지적 남은 지적 새로 생긴 지적

매번 새로 답하는 1회성 도구가 아니라, 김나영이 무엇을 고쳤고 어떤 지적이 해결됐는지를 기억해 점수가 오르는 과정을 함께 추적하는 개인 맞춤형 피드백 루프다.

"주제도 못 정한 막막한 상태"에서 시작해서, ①공모전 분석 → ②아이디어 발굴 회의 → ③아이디어 구체화 회의 → ④기획서 작성 → ⑤문서 완성도 자동 점검 → ⑥버전 추적형 완성 리포트로 반복 개선까지 — 기획서 작성의 처음부터 끝까지를 한 서비스 안에서 끝낼 수 있게 하는 것이 AI Review Board의 전체 그림입니다.
